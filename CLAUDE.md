# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
pnpm dev          # Start dev server on port 3000
pnpm build        # Production build
pnpm test         # Run tests with Vitest
pnpm lint         # Biome lint
pnpm format       # Biome format
pnpm check        # Biome lint + format check

# Database (requires .env.local with DATABASE_URL)
pnpm db:generate  # Generate Prisma client
pnpm db:push      # Push schema changes to DB (no migration file)
pnpm db:migrate   # Create and apply a migration
pnpm db:studio    # Open Prisma Studio
pnpm db:seed      # Run seed script
```

Run a single test file: `pnpm vitest run src/path/to/file.test.ts`

## Architecture

**Framework**: TanStack Start (React SSR framework built on TanStack Router + Vite + Nitro)

**Routing**: File-based routing under `src/routes/`. The auto-generated `src/routeTree.gen.ts` must not be edited manually â€” it's regenerated by the Vite plugin on changes to route files.

- `src/routes/__root.tsx` â€” root layout (Header, Footer, theme init script, devtools)
- `src/routes/index.tsx` â€” home page
- `src/routes/api/auth/$.ts` â€” Better Auth catch-all API handler

**Data layer**:
- `src/db.ts` â€” Prisma client singleton (PostgreSQL via `@prisma/adapter-pg`), uses `globalThis.__prisma` to prevent multiple instances in dev
- `prisma/schema.prisma` â€” schema; Prisma client is generated to `src/generated/prisma/`
- All DB commands require `DATABASE_URL` in `.env.local`

**Auth**: Better Auth (`src/lib/auth.ts` server config, `src/lib/auth-client.ts` client config). Email/password enabled. Uses `tanstackStartCookies` plugin for SSR cookie handling.

**Environment variables**: Validated with T3Env in `src/env.ts`. Client vars must be prefixed `VITE_`. Import as `import { env } from "#/env"` (the `#/` alias maps to `src/`).

**Styling**: Tailwind CSS v4 via Vite plugin. Component utilities in `src/lib/utils.ts` (clsx + tailwind-merge). Shadcn components added via `pnpm dlx shadcn@latest add <component>`.

**Linting/formatting**: Biome with tab indentation and double quotes for JS/TS. Only processes files under `src/` and a few config files; `src/routeTree.gen.ts` and `src/styles.css` are excluded.

## Key patterns

- **Server functions**: Use `createServerFn` from `@tanstack/react-start` for server-side logic called from components
- **Path alias**: `#/` resolves to `src/` (configured in `package.json` `imports` field)
- **Theme**: Dark/light/auto theme stored in `localStorage`, applied via inline script in `<head>` to prevent FOUC
